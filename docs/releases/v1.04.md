# Guderian 1.04 Release Notes

## Synopsis

Guderian 1.04 makes the campaign's enjoyment model explicit and testable. The 35-battle chronology is now audited through the new `GuderianFun` SwiftPM target, which scores clarity, agency, challenge, mastery, novelty, consequence, pacing, and ethical framing against the fun model in `fun/FUN.md`. A companion `funDT` view reconstructs every battle as a temporal-difference profile, showing how much each scenario changes player posture, terrain systems, objectives, pressure, force texture, tempo, and command scope compared with earlier battles. The result is a sharper release-readiness lens: all battles still pass the core fun gates, while the late-war staff-context stretch is now visibly flagged as strong but more temporally repetitive.

For players, this release protects the sober 35-battle playable campaign by checking that each battle remains readable, purposeful, and historically grounded. The report confirms that live unified scenario flows still complete across the full campaign, and it identifies where future scenario work should add more texture rather than simply more content. For downstream source users, 1.04 adds the compiled `GuderianFun` library, the `GuderianFunReport` executable, focused fun/funDT tests, and updated release-version plumbing while preserving the local embedded `dzw` dependency and the packaging flow in `RELEASE.md`.
