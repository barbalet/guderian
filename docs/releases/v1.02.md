# Guderian 1.02 Release Notes

## Synopsis

Guderian 1.02 focuses on making the campaign easier to evaluate, teach, and release. Players now get a cleaner first-run teaching flow and first-game button guidance: contextual hover dialogs explain battle controls the first time they are encountered, persist dismissal after the player has completed the first game, and stay disabled for the automated test harness. The README has been tightened around support channels, Discord, GitHub issue reporting, and cleaner initial battle-map screenshots for Tuchola Forest and Bialystok-Minsk, showing starting unit positions without tactical overlay clutter. The release also incorporates the local `REVIEW.md` hardening pass: legacy `TE`/`te` naming has been moved to `DZW`/`dzw` conventions, shared board dimensions and scenario bridge ownership are better documented, persistence and automation stage contracts have stronger typed coverage, and failed or no-op battle actions are surfaced more clearly in tests and UI behavior. For downstream source users, 1.02 keeps the Guderian-specific changes local to this checkout and its embedded `dzw` source, with release instructions updated to treat the version number as an input, tag the exact release commit, and carry a concise synopsis with the package.
